Interactivity in this Template

When you opt for the Global interactivity location, render modes are defined in App.razor and apply to all child components. Providers are globally configured in the MainLayout.

If you choose Per page/component interactivity, the following providers must be included on each interactive page:

<MudPopoverProvider />

<MudDialogProvider />

<MudSnackbarProvider />

Pages without a specified render mode default to Server-Side Rendering (SSR). While MudBlazor supports SSR, interactive components such as buttons and menus will not function unless interactivity is enabled.

What’s New in Blazor (.NET 9)
Prerendering

In .NET 9, every Blazor page is prerendered on the server regardless of the selected render mode.

This means all required services must be available server-side, even when using WebAssembly.

Prerendering improves perceived performance and initial load times.
